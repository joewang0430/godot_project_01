# Godot Lab 01 - 阶段性开发记录

## 当前阶段目标（学习优先）

以下内容用于防止聊天上下文丢失，作为本项目后续开发的统一基线。

1. 保持现有玩家移动系统不变（当前 FSM 移动逻辑不改）。
2. 先实现玩家拿枪攻击，再实现敌人系统。
3. 攻击逻辑：固定攻速，枪口朝向始终为“玩家到鼠标”的方向。
4. 敌人生成：每局开始时随机刷在地图地面上若干个。
5. 敌人行为：仅近战，不远程；沿 X 轴方向逼近玩家。
6. 伤害规则：敌人接触玩家时按固定频率造成伤害。
7. 玩家死亡：血量归零即死亡。
8. 击杀收益：玩家每击杀 1 个敌人获得 1 金币。
9. 胜利条件：地图内敌人被清空。

## 实施策略（分步学习，不跳步）

### Step 1：先搭最小可运行骨架（先做这个）

1. 在玩家节点下添加：
   - `GunPivot` (`Node2D`)
   - `Muzzle` (`Marker2D`，作为枪口，挂在 `GunPivot` 下)
2. 创建 `Bullet` 场景：
   - 根节点 `Area2D`
   - 子节点 `CollisionShape2D`
   - 可选 `Sprite2D`
3. 创建 `Enemy` 场景：
   - 根节点 `CharacterBody2D`（建议）
   - 子节点 `CollisionShape2D`
   - 可选 `Sprite2D`
4. 在 Project Settings 创建分组：
   - `player`
   - `enemy`
5. 先把玩家加入 `player` 组。

### Step 2：第一段代码目标

1. 枪始终朝鼠标方向旋转。
2. 固定攻速开火。
3. 先打通“发射子弹”，暂不接入敌人受伤。

## 验收标准（当前阶段）

1. 场景结构可见：玩家下有 `GunPivot` 和 `Muzzle`。
2. 已存在可实例化的 `Bullet` 和 `Enemy` 场景。
3. 分组 `player`、`enemy` 已创建，玩家已进 `player` 组。
4. 运行时可观察枪口跟随鼠标并按固定频率发射。
